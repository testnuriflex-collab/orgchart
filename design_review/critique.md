# 조직도 Studio — 사용자 관점 무자비 비평서

> 페르소나: 방금 유료로 다운받아 처음 실행한 **인사담당자**(비개발자) + 20년차 프로덕트 디자이너.
> 기준: "이 돈이 아깝지 않은가?" 첫인상 3초, 정보 밀도, 가독성, 온보딩.
> 근거: 실제 오프스크린 렌더 스크린샷 3종(`main.png`, `roster.png`, `empty.png`, 1480×940).

---

## 1차 비평 (before)

### 🔴 치명 (Blocker) — 이 상태면 환불 요청감

**B1. 조직도가 캔버스 한복판에 콩알만 하게 뭉쳐 있다 — 첫 화면이 고장 난 것처럼 보인다.**
`main.png`을 열면 조직도가 캔버스 폭의 약 30%만 차지한 채 중앙 하단에 초소형으로 박혀 있고,
위·아래로 화면 절반이 텅 빈 회색 여백이다. 카드 안 글자는 1~2px 수준으로 **완전히 읽을 수 없다**.
- 원인: `fit_chart()`가 `fitInView(KeepAspectRatio)`로 전체 씬(양옆 96px 패딩 포함)을 화면에 억지로 맞춘다.
  16개 팀이 가로로 늘어선 넓은 트리(폭 ≈ 4,000px+)를 872px 캔버스에 맞추니 배율이 ~0.2x로 떨어지고,
  종횡비 유지 탓에 세로로는 화면 중앙에 뜨면서 위아래가 거대한 공백이 된다.
- 사용자 체감: "다운받자마자 깨진 화면. 데이터는 들어갔다는데 뭘 봐야 하는지 모르겠다." → **신뢰 즉시 붕괴.**
- 기대: 처음부터 카드 글자가 읽히는 배율로, 캔버스를 꽉 채워 상단 중앙 정렬. 큰 조직은 스크롤/줌으로 탐색.

**B2. 캔버스 "조직도 / 현재 보고 체계" 헤딩이 축소에 휩쓸려 존재감이 없다.**
루트 카드 위에 붙은 씬 내부 타이틀은 축소되면 먼지처럼 사라진다. 캔버스 문맥 헤더는 뷰(고정) 레이어여야 한다.

### 🟠 심각 (Major)

**M1. 빈 상태(empty)가 성의 없다 — 처음 켠 사람을 방치한다.**
`empty.png`: 캔버스 중앙에 흐린 회색 텍스트 3줄만 덩그러니. 아이콘·일러스트 없음, **행동 유도(CTA) 버튼이 캔버스 안에 없다**(가져오기는 헤더 구석 아이콘일 뿐). 좌측 목록도 흰 공백.
처음 실행한 인사담당자는 "그래서 뭘 누르라고?"가 된다. 1-2-3 온보딩·큰 버튼이 있어야 한다.

**M2. 표 편집 화면에서 마지막 컬럼들이 잘린다.**
`roster.png`: 이메일 컬럼은 과도하게 넓고, 오른쪽 직책·재직상태·입사일·퇴사일은 잘려 가로 스크롤을 강요한다.
`resizeColumnsToContents()`가 컬럼 폭 균형을 깨뜨린 결과. 표가 컨테이너 폭을 다 쓰지 못한다.

**M3. 우측 패널이 맥락과 어긋난다.**
표 편집 화면인데 우측에는 조직도 전용 "표시 항목/속성/조직명 편집"이 그대로 떠 있다. 빈 상태에서도 마찬가지로 데이터 0인데 토글·속성 폼이 노출돼 첫 실행자를 혼란스럽게 한다.

### 🟡 개선 (Minor)

- **m1.** 좌측 조직 목록이 위계 없는 평면 나열이라 "경영지원본부·0명"처럼 상위 조직이 0명으로 보여 오해를 부른다(직속만 카운트). 들여쓰기 또는 소계 표기 필요.
- **m2.** 헤더 툴바가 아이콘만이라(가져오기 제외) 처음 쓰는 사람은 기능을 hover 전엔 모른다.
- **m3.** 캔버스 배경(#F1F3F6)과 패널 배경(#EDEFF3) 대비가 약해 조직도 영역 경계가 흐릿하다.

## 1차 재작업 내역 → 2차 비평 (after)

### 재작업(코드 수정)
- **B1 해결**: `fit_chart()`를 둘로 분리. 첫 화면·데이터 갱신은 새 `reset_view()`가 담당 —
  가독 하한(`MIN_READABLE_ZOOM=0.72`) 아래로는 축소하지 않고, 트리가 크면 그 배율로 고정해 **상단 중앙**에
  루트부터 정렬(이후 스크롤/줌 탐색). '전체 보기' 버튼만 종전의 전체 맞춤(overview)을 유지.
  `OrgChartViewMixin`에 `showEvent`/`resizeEvent` 훅을 달아 실제 앱에서도 첫 표시·리사이즈 시 자동 재정렬(사용자가 직접 줌하면 중단).
- **M1 해결**: 캔버스 빈 상태를 실제 위젯 페이지(`_build_empty_state`)로 교체 — 뱃지 아이콘 + "조직도를 시작해 볼까요?" +
  1·2·3 온보딩 스텝 + **실제 CTA 버튼**(인사 파일 가져오기 / 표준 템플릿 받기).
- **M2 해결**: 표 컬럼 리사이즈를 `ResizeToContents` + 이메일 `Stretch`로 재정의(+행 높이 38). 뒤 컬럼 잘림·강제 가로 스크롤 제거, 11개 컬럼 모두 노출.
- **M3 해결**: 우측 패널을 맥락 인지형으로 — 조직도 화면에서만 노출, 표 편집·빈 상태에서는 숨김.
- **m1 해결**: 좌측 조직 목록을 **위계(DFS) 순서 + 들여쓰기 + 하위 포함 인원수**로 재구성.
  본부가 "0명"으로 오해되던 문제 제거(경영지원본부 3명·사업본부 5명·제품본부 6명으로 정확 표기).

### 2차 비평(새 눈으로 재심사, after_final.png / roster.png / empty.png)
- **첫인상 3초**: ✅ 통과. 첫 화면에 회사 루트 카드와 본부·팀·구성원이 또렷하게 읽힌다. "고장 났나?" 반응 소멸.
- **가독성**: ✅ 카드 제목·직급·직책·사번·재직상태 모두 판독 가능. 재직=녹색, 퇴사=회색 바로 상태 즉시 구분.
- **온보딩**: ✅ 데이터 0일 때 무엇을 할지 버튼과 3스텝으로 명확.
- **정보 밀도/위계**: ✅ 좌측 목록이 조직 구조와 규모를 한눈에 전달.
- 잔여 minor: 표의 이메일 셀은 긴 주소가 말줄임(…)될 수 있음(셀 편집·툴팁으로 전체 확인 가능) — 실사용에 지장 없음.
  헤더 아이콘 라벨(m2)·캔버스 대비(m3)는 hairlineStrong 테두리로 경계 강화까지만 반영, 치명도 낮아 수용.

## 판정 (2차)
**✅ 상용 제품 수준 — "돈 주고 산 제품"으로 납득 가능.** 첫 화면에서 조직도가 제대로 읽히고,
빈 상태·표 편집·목록 위계까지 맥락이 일관된다. 1차의 환불급 결함(B1)과 주요 결함(M1~M3, m1) 모두 해소.

## 3차 재작업 (부모 직접 검수 후 잔여 결함 3건)

부모가 after_final.png를 확대 검수해 첫 화면 정렬 결함 3건 추가 확인 → 재작업:

- **C1 잘린 카드 노출**: 좌우 엣지에 반쯤 클리핑된 카드가 보이던 문제. 고정 폭 스크롤 스냅으로는
  넓은 트리에서 양 끝을 동시에 카드 틈에 맞출 수 없어 접근을 전환 — 루트를 포함하며 좌우 끝이
  모두 '카드 틈'에 떨어지는 수평 구간을 골라 그 구간을 뷰포트 폭에 정확히 맞춘다
  (`_clean_cut_positions` + `_best_clean_span`). 배율은 가독 범위(0.72~1.05) 안에서 유지되고
  좌우 엣지가 카드 경계와 일치해 **잘린 카드 0개**. 구간 선택은 루트를 화면 중앙에 두도록 점수화.
- **C2 씬 타이틀 충돌**: 씬 내부 "조직도/현재 보고 체계" 텍스트 제거 → 뷰포트 고정 좌상단
  오버레이 라벨(`#canvasTitle`)로 이전. 스크롤·축소와 무관, 루트 카드와 충돌 없음.
- **C3 상단 공백**: 트리가 뷰포트보다 짧을 때 `centerOn`이 세로 중앙 배치를 강제하던 문제.
  씬 하단을 필요한 만큼 넓혀 상단 정렬이 실제 적용되게 하고, 트리 최상단을 뷰포트 상단 ~32px에 정렬.

### 3차 확인(after_final.png)
- 잘린 카드 0개(좌측 사업전략팀·우측 운영기획팀·상단 루트 모두 온전히 노출). ✅
- 타이틀 충돌 0(좌상단 고정 pill). ✅
- 상단 공백 해소(루트 카드가 캔버스 상단 ~32px). ✅

## 판정 (3차)

## 4차: 인터랙션 + 블루 테마 전환 후 무자비 재비평

사용자 추가 요구(휠 줌·블루 색상)를 반영한 뒤, "방금 다운받은 인사담당자" 관점으로 인터랙션 감각까지 재심문.
근거: `after_final.png`(블루 첫 화면), `zoom.png`(휠 줌 인 상태), `roster.png`, `empty.png`.

### 반영 내역
- **인터랙션(휠 줌·패닝)**: 보조키 없이 마우스 휠만으로 커서 기준 줌(AnchorUnderMouse), Ctrl+휠도 동일.
  배율 한계 0.1x~3x. 씬 rect를 콘텐츠보다 넉넉히(가로 ≥1400·세로 ≥900px, 콘텐츠 75% 추가) 확장해
  줌·패닝이 경계에 막히지 않는 무한 캔버스 감각. 스크롤은 빈 공간 드래그(손바닥 패닝, ScrollHandDrag)로 대체.
  '전체 보기' 버튼은 패닝 여백을 제외한 실제 콘텐츠 경계에 맞추도록 수정.
- **블루 색상 체계**: 토큰 레벨에서 버건디→블루 전면 교체(accent #1D4ED8, hover/active/soft 파생색).
  하드코딩 잔재 전수 스캔 후 chart_view 상수까지 교체 — 회사=딥 네이비(#12294A), 본부=블루(#1D4ED8),
  팀=슬레이트(#64748B), 재직 뱃지·인디케이터=블루, 커넥터=뉴트럴 블루그레이. 검색 하이라이트도 블루 틴트.
  대비: 블루 버튼 위 흰 텍스트 ~5.5:1(WCAG AA), 네이비 루트 위 흰 텍스트 초과 충족.

### 4차 확인
- **줌 감각**: ✅ 휠 인/아웃이 15% 스텝으로 부드럽고, 확대 시 카드·텍스트가 선명(`zoom.png`).
- **커서 피드백**: ✅ 캔버스는 손바닥 커서(패닝 유도), 버튼은 포인팅 핸드, hover 시 배경/보더 변화(QSS).
- **좌측 엣지 슬리버**: ✅ 잔존 0 — 뷰포트 좌우가 카드 '틈'에 정확히 맞아 반쯤 잘린 카드가 없다.
- **색 일관성**: ✅ 버건디 잔재 0(코드·테스트 전수 스캔). 레벨 위계가 네이비→블루→슬레이트로 또렷.
- **명단 표·빈 상태**: ✅ 블루로 통일, 잘림·맥락 문제 없음.

## 판정 (2차)
**✅ 상용 제품 수준 — "돈 주고 산 제품"으로 납득 가능.** 블루 브랜딩이 일관되고, 휠 줌·드래그 패닝으로
탐색이 자연스러우며, 첫 화면은 잘린 카드 없이 읽힌다.

## 5차: 실사용 버그 — 드래그 시 연결선이 옛 좌표에 박힘 (근본 수정)

**증상**: 노드 카드를 드래그하면 카드는 움직이는데 부모-자식 연결선이 원래 자리에 남아 어긋남.
**근본 원인**: 연결선이 레이아웃 시점 좌표로 `addLine` 정적 아이템으로 한 번만 그려져 노드 이동을 추적하지 못함.
**수정**:
- 연결선을 노드 참조 기반 동적 아이템으로 전환(`OrgConnectorController`). 부모 그룹당 QGraphicsPathItem 하나를
  두고, 노드의 '현재' 위치(레이아웃 좌표 + 아이템 pos)에서 직교 경로를 재계산.
- 움직이는 카드(조직·직원)에 `ItemSendsGeometryChanges` 부여 + `itemChange(ItemPositionHasChanged)`에서
  컨트롤러 `update()` 호출 → 드래그 중 실시간 재작도.
- 드롭 일관화: 유효 타겟이면 이동 확정 후 재레이아웃, 무효(빈 공간 등)면 재레이아웃으로 **원위치 스냅백**.
  클릭과 드래그를 이동 임계(6px)로 구분해 단순 클릭(선택)은 건드리지 않음.
- 전수 스캔: 뱃지·라벨·인디케이터 등 카드 내부 요소는 모두 카드(그룹)의 자식이라 카드와 함께 이동 —
  좌표 고정으로 남는 요소는 연결선뿐이었고 이를 근본 수정. 캔버스 타이틀은 이미 뷰 고정 오버레이.

### 5차 확인
- 회귀 테스트: 헤드리스에서 자식 노드 `setPos` → 연결선 경계가 새 중심 x까지 재계산됨을 검증(자동 트리거). ✅
- 스크린샷(`drag.png`): 세일즈팀 카드를 이동하니 부모→팀 버스와 팀→구성원(문태오) 선이 새 위치로 따라옴. ✅

## 판정 (최종)
**✅ 상용 제품 수준.** 드래그 중에도 선-카드가 정합을 유지하고, 무효 드롭은 스냅백된다.
전체 테스트 82건 통과, ruff clean.
